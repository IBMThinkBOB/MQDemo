**Hands-On Demo: Building and Troubleshooting IBM MQ on LinuxONE Using Bob**

**Summary**

This demo introduces attendees to LinuxONE by walking through a simple, hands-on exercise using IBM MQ and Bob. Participants will learn how to set up a basic messaging system, interact with it, and troubleshoot a real issue using AI guidance. The focus is on building confidence, demonstrating practical workflows, and showing how Bob can accelerate learning in a new technical environment.

**Demo Flow Overview**

- Connect to a LinuxONE environment
- Use Bob to guide IBM MQ installation and setup
- Create and start a queue manager
- Define and use a messaging queue
- Send and receive a test message
- Introduce a controlled issue
- Use Bob to diagnose and resolve the issue
- Validate the working system

**Step-by-Step Walkthrough**

**Step 1: Environment Introduction**

Participants log into a pre-provisioned LinuxONE environment and are briefly oriented to the interface and workflow.

The focus here is not on deep technical detail, but on helping attendees feel comfortable navigating a new system. They will see:

- How to access the environment
- What a typical working session looks like
- Where they will run commands and interact with tools

**Step 2: Guided Setup with Bob**

Participants are introduced to Bob and how it functions as an AI assistant within their workflow.

Rather than following static documentation, attendees actively:

- Ask Bob how to install and set up IBM MQ
- Use Bob to interpret instructions and explain unfamiliar concepts
- Follow guided steps generated by Bob
- Have Bob run terminal commands for installation

The goal is to demonstrate that users do not need prior expertise, and can rely on Bob to guide them through the process while learning along the way.

**Step 3: Queue Manager Creation**

Participants create and start a queue manager.

During this step, attendees:

- Learn what a queue manager is and why it is needed
- Use Bob to understand the purpose of each step
- See how a core service is initialized and made available

**Step 4: Queue Creation**

Participants define a local queue that will hold messages.

In this step, they:

- Understand the role of a queue in message-based systems
- Create a simple queue to enable communication
- Use Bob to explain terminology and configuration choices

**Step 5: Message Flow Demonstration**

Participants send and receive a message using the queue they created.

They will:

- Place a message into the queue
- Retrieve the message from the queue
- Confirm that messaging is working end-to-end

**Step 6: Introduce a Controlled Failure**

A simple, intentional issue is introduced into the environment to simulate a real-world problem.

This could include

- Stopping the queue and sending a message
- Users interact with a queue name that hasn't been created
- Permission/Access denied issue
- Misconfigured but running service

Participants observe:

- That the previously working system no longer behaves as expected
- Error output or failed operations

The goal is to create a realistic scenario where troubleshooting is required, without adding unnecessary complexity.

**Step 7: Troubleshooting with Bob**

Participants use Bob to diagnose and resolve the issue. They will:

- Provide context or error information to Bob
- Receive explanations of what went wrong
- Follow recommended steps to fix the issue

This step highlights Bob's strongest value: accelerating problem-solving and reducing the need to manually search through documentation.

**Step 8: Validation**

Participants apply the fix and re-run the earlier message flow.

They confirm:

- The system is working again
- The issue has been successfully resolved

**Takeaways**

- Participants build and interact with a real messaging system on LinuxONE
- Attendees understand the basic components of IBM MQ (queue manager, queue, message flow)
- Bob serves as a practical assistant for learning, setup, and troubleshooting
- Troubleshooting is simplified through AI-guided explanations and suggestions
- LinuxONE is approachable and usable for hands-on development and experimentation
- Attendees leave with the confidence to continue learning independently using their provided environment access
