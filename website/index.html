📋 Lab Overview

Welcome! In this hands-on lab, you will:

  • Connect to your dedicated LinuxONE virtual machine
  • Download and install IBM MQ Advanced for Developers
  • Create and configure a queue manager
  • Define a message queue
  • Send and receive your first message
  • Validate a working enterprise messaging system

Time Required: 45-60 minutes
Difficulty: Beginner-friendly
Prerequisites: Basic Linux command-line knowledge

Each participant receives:

  • Dedicated LinuxONE VM - Your own isolated environment

  • SSH Key Pair - Secure authentication

✅ Before You Begin

Step 1

Connect to Your LinuxONE VM

1.1 Prepare Your SSH Key

First, set the correct permissions on your SSH key file:

chmod 600 /path/to/your-key-file.pem

Note: Replace /path/to/your-key-file.pem with the actual path to your downloaded key file.

1.2 Connect via SSH

Use the following command pattern with your personalized details:

ssh -i /path/to/your-key-file.pem username@your-server-ip

Example:

ssh -i ~/.ssh/linuxone-lab.pem student01@192.168.1.100

1.3 First Connection

On your first connection, you may see a host key verification prompt:

The authenticity of host '192.168.1.100' can't be established.
Are you sure you want to continue connecting (yes/no)?

Type yes and press Enter.

✅ Success! You should now see a Linux command prompt like:
[username@hostname ~]$
Step 2

Validate Your Environment

2.1 Verify System Information

Run these commands to confirm you're on the correct system:

hostname

Expected: Your VM's hostname

uname -a

Expected: Linux system information showing s390x architecture

whoami

Expected: Your username

2.2 Verify Sudo Access

sudo -v

Expected: No errors (may prompt for password)

2.3 Check Internet Connectivity

ping -c 3 ibm.com

Expected: Successful ping responses

✅ Checkpoint: If all commands succeed, your environment is ready for IBM MQ installation!
⚠️ Troubleshooting: If any command fails, contact event support before proceeding.
Step 3

Download and Install IBM MQ

3.1 Create Working Directory

mkdir -p ~/mq-install
cd ~/mq-install

3.2 Download IBM MQ Advanced for Developers

Download the latest IBM MQ for Linux on s390x:

# Download IBM MQ 9.4 for Linux s390x
wget https://public.dhe.ibm.com/ibmdl/export/pub/software/websphere/messaging/mqadv/mqadv_dev94_linux_s390x.tar.gz

Note: This download is approximately 500MB and may take a few minutes.

3.3 Extract the Archive

tar -xzf mqadv_dev94_linux_s390x.tar.gz

3.4 Accept the License

cd MQServer
sudo ./mqlicense.sh -accept

3.5 Install IBM MQ

# Install all MQ components
sudo rpm -ivh MQSeries*.rpm

Note: Installation takes 2-3 minutes. You'll see progress for each component.

3.6 Set Up MQ Environment

# Add MQ to your PATH
echo 'export PATH=$PATH:/opt/mqm/bin' >> ~/.bashrc
source ~/.bashrc

3.7 Verify Installation

dspmqver
Expected Output: 
Name:        IBM MQ
Version:     9.4.0.0
...
InstPath:    /opt/mqm
✅ Success! IBM MQ is now installed on your LinuxONE VM!
Step 4

Create and Configure Queue Manager

4.1 Create MQ User and Group

# Create mqm group and user
sudo groupadd mqm
sudo useradd -g mqm -d /var/mqm mqm
sudo passwd mqm

Note: Set a password when prompted (e.g., "mqm123")

4.2 Add Your User to MQ Group

sudo usermod -a -G mqm $USER

4.3 Set MQ Permissions

sudo chown -R mqm:mqm /var/mqm
sudo chmod -R 775 /var/mqm

4.4 Create Queue Manager

sudo -u mqm crtmqm QM1
Expected Output: 
IBM MQ queue manager created.
Directory '/var/mqm/qmgrs/QM1' created.
...
IBM MQ queue manager 'QM1' created.

4.5 Start Queue Manager

sudo -u mqm strmqm QM1
Expected Output: 
IBM MQ queue manager 'QM1' starting.
...
IBM MQ queue manager 'QM1' started.

4.6 Verify Queue Manager Status

sudo -u mqm dspmq
Expected Output: 
QMNAME(QM1)                                               STATUS(Running)
✅ Success! Your queue manager QM1 is now running!
Step 5

Create Queue and Test Messaging

5.1 Create a Local Queue

sudo -u mqm bash -c "echo 'DEFINE QLOCAL(EVENT.Q1) REPLACE' | runmqsc QM1"
Expected Output: 
5724-H72 (C) Copyright IBM Corp. 1994, 2024.
Starting MQSC for queue manager QM1.

     1 : DEFINE QLOCAL(EVENT.Q1) REPLACE
AMQ8006I: IBM MQ queue created.

5.2 Verify Queue Exists

sudo -u mqm bash -c "echo 'DISPLAY QLOCAL(EVENT.Q1)' | runmqsc QM1"

5.3 Send Your First Message

echo "Hello from LinuxONE!" | sudo -u mqm /opt/mqm/samp/bin/amqsput EVENT.Q1 QM1
Expected Output: 
Sample AMQSPUT0 start
target queue is EVENT.Q1
Sample AMQSPUT0 end

5.4 Receive the Message

sudo -u mqm /opt/mqm/samp/bin/amqsget EVENT.Q1 QM1
Expected Output: 
Sample AMQSGET0 start
message 
no more messages
Sample AMQSGET0 end
🎉 Congratulations! You've successfully sent and received your first message on IBM MQ!

Validation and Experiments

6.2 Experiment: Multiple Messages

Try sending multiple messages and observe FIFO ordering:

echo "Message 1" | sudo -u mqm /opt/mqm/samp/bin/amqsput EVENT.Q1 QM1
echo "Message 2" | sudo -u mqm /opt/mqm/samp/bin/amqsput EVENT.Q1 QM1

View messages without removing them from the queue:

🔧 Troubleshooting

❌ Cannot connect via SSH

  • Verify key file permissions: chmod 600 your-key-file.pem
  • Check server IP/hostname is correct
  • Verify VPN connection if required
  • Contact event support for connection details verification

❌ Download fails or is very slow

  • Check internet connectivity: ping ibm.com
  • Try alternative download mirror if provided
  • Contact facilitator for pre-staged installation files

❌ Installation fails

  • Verify sudo access: sudo -v
  • Check disk space: df -h
  • Review error messages carefully
  • Contact event support with error output

❌ Queue manager won't start

  • Check if already running: sudo -u mqm dspmq
  • View error logs: sudo cat /var/mqm/qmgrs/QM1/errors/AMQERR01.LOG
  • Verify mqm user exists: id mqm
  • Check permissions: ls -la /var/mqm

❌ Cannot send/receive messages

  • Verify queue manager is running: sudo -u mqm dspmq
  • Check queue exists: sudo -u mqm bash -c "echo 'DISPLAY QLOCAL(EVENT.Q1)' | runmqsc QM1"
  • Verify user permissions: groups (should include mqm)
  • Try recreating the queue

📚 Quick Reference

Queue Manager Commands

# Check status
sudo -u mqm dspmq

# Start queue manager
sudo -u mqm strmqm QM1

# Stop queue manager
sudo -u mqm endmqm QM1

Queue Operations

# Create queue
echo 'DEFINE QLOCAL(MYQUEUE)' | sudo -u mqm runmqsc QM1

# Display queue
echo 'DISPLAY QLOCAL(MYQUEUE)' | sudo -u mqm runmqsc QM1

# Check depth
echo 'DISPLAY QLOCAL(MYQUEUE) CURDEPTH' | sudo -u mqm runmqsc QM1

Messaging Commands

# Send message
echo "text" | sudo -u mqm /opt/mqm/samp/bin/amqsput QUEUE QM1

# Receive message
sudo -u mqm /opt/mqm/samp/bin/amqsget QUEUE QM1

# Browse messages
sudo -u mqm /opt/mqm/samp/bin/amqsbcg QUEUE QM1

System Commands

# Check MQ version
dspmqver

# View error logs
sudo cat /var/mqm/qmgrs/QM1/errors/AMQERR01.LOG

# Check disk space
df -h

🎓 What You've Learned

Core Concepts

  • Queue Manager - The MQ service that manages queues and routes messages
  • Queue - A storage location for messages (like a mailbox)
  • Message - Data sent between applications
  • PUT - Sending a message to a queue
  • GET - Retrieving and removing a message from a queue
  • BROWSE - Viewing messages without removing them
  • FIFO - First In, First Out message ordering
